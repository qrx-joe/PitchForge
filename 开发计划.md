## 🔄 实际交付与原计划对比

### 一致 ✅
- 前端: 原生 HTML/CSS/JS (零构建)
- 后端: Node.js + Express
- 字段: 5 输入 / 4 输出
- 限流: 15 min / 30 次
- 备用输出: 1 份预生成

### 调整 🔧
- **AI 集成**: 5 节点工作流 → **单次 Application Call + 聚合 Prompt**
  - 理由: MVP 上线速度, 调试链路短
  - 预留: 4 个独立 Prompt 文件已就位, P1 升级零代码改动
- **品牌**: "百炼 CLI" / "通义千问" → "阿里云 AI 能力"
  - 理由: 品牌中性化, P2 多模型扩展
  - 变量: 保留 `BAILIAN_*` 命名 (向后兼容)
- **测试**: 手动 ✅ / 自动化 ⏳
  - 理由: 90 分钟内, 手动测试覆盖 80% 场景
  - 计划: P1 补 Jest + Playwright

### 新增 🆕
- **Mock 兜底** (`mockData.js`) - 原计划没明确, 实施时加的, 救命的
- **业务错误码体系** (`backend/services/bailianService.js`) - 原计划用 HTTP 状态码, 实施时升级
- **备用输出文件** (`docs/sample-output.md`) - 路演容灾必做
- **6 个评委问答** - 1 个技术深度备用是现场追问时加的

## 🚨 风险与应对 (路演现场)

| 风险 | 概率 | 影响 | 应对措施 | 状态 |
|------|------|------|----------|------|
| 百炼 API 调用失败 | 中 | 高 | 切换到 `docs/sample-output.md` 备用 | ✅ 已准备 |
| 网络完全断开 | 中 | 高 | 同上 (本地 Markdown 直接展示) | ✅ 已准备 |
| 输出格式异常 | 低 | 中 | 重新点击生成 | ✅ 前端支持 |
| 评委要求现场改输入 | 中 | 中 | 接受并演练过 | ✅ 话术已备 |
| 评委技术追问太深 | 中 | 中 | 诚实回答 + 引到 Q6 备用 | ✅ 话术已备 |
| 输出超过 90 秒 | 低 | 高 | 控制进度条到 100% 后立即说"完成" | ✅ 已演练 |
