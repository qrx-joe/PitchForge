### 2.3 色彩规范

基于 PRD 定义的色彩系统，应用规范如下：

| 色彩角色 | 色值 | 应用场景 | 对比度要求 |
|----------|------|----------|------------|
| 背景色 | #F7F8FA | 页面整体背景 | - |
| 主色 | #4F46E5 | 按钮、链接、强调元素 | WCAG AA (4.5:1) |
| 强调色 | #10B981 | 成功状态、高亮、完成标记 | WCAG AA (4.5:1) |
| 文本色 | #111827 | 标题、正文 | - |
| 辅助文本 | #6B7280 | 提示、说明、占位符 | WCAG AA (4.5:1) |
| 卡片背景 | #FFFFFF | 内容区块背景 | 需要阴影区分层级 |

**色彩使用原则**:
1. 主色仅用于 **可交互元素** (按钮、链接)
2. 强调色仅用于 **成功/完成状态**
3. 避免在同一个页面使用超过 3 种颜色
4. 确保文字与背景的对比度符合 WCAG 2.0 AA 标准

### 3.2 命名规范

#### 文件命名
- **HTML/CSS/JS**: `kebab-case` (例: `style.css`, `app.js`)
- **组件**: `PascalCase` (例: `InputForm.js`)
- **测试文件**: `*.test.js` 或 `*.spec.js`

#### 变量/函数命名
- **变量**: `camelCase` (例: `projectIdea`, `targetUser`)
- **常量**: `UPPER_SNAKE_CASE` (例: `API_BASE_URL`, `MAX_INPUT_LENGTH`)
- **函数**: `camelCase` (例: `generatePitch()`, `handleClick()`)
- **类**: `PascalCase` (例: `BailianService`, `GenerateController`)
- **私有变量**: `_camelCase` (例: `_apiKey`, `_timeout`)

#### CSS 类命名
- **BEM 规范**: `block__element--modifier`
  - 例: `.input-form__field--error`
  - 例: `.btn--primary`, `.btn--secondary`

### 3.3 代码格式

**缩进**: 2 空格 (HTML/CSS/JS 统一)

**换行符**: LF (Unix 风格)

**最大行宽**: 100 字符

**引号**: 单引号 (JS), 双引号 (HTML)

**分号**: 必须 (JS)

**尾随逗号**: 多行对象/数组最后一项加逗号

## 四、API 设计规范

### 4.1 RESTful 原则

1. **资源导向**: 使用名词表示资源
   - ✅ `/api/generate` (生成资源)
   - ❌ `/api/doGenerate` (动词)

2. **HTTP 方法语义**:
   - `GET`: 获取资源
   - `POST`: 创建资源
   - `PUT`: 更新资源
   - `DELETE`: 删除资源

3. **统一响应格式**:
   ```json
   {
     "success": true,
     "data": { ... },
     "message": "操作成功",
     "timestamp": "2026-06-11T00:00:00Z"
   }
   ```

### 4.2 错误处理规范

**错误响应格式**:
```json
{
  "success": false,
  "error": {
    "code": "INVALID_INPUT",
    "message": "输入参数无效",
    "details": "project_idea 不能为空"
  },
  "timestamp": "2026-06-11T00:00:00Z"
}
```

**常见错误码**:
| 错误码 | HTTP 状态码 | 说明 |
|--------|-------------|------|
| INVALID_INPUT | 400 | 输入参数无效 |
| UNAUTHORIZED | 401 | 未授权 (API Key 错误) |
| RATE_LIMITED | 429 | 请求过于频繁 |
| INTERNAL_ERROR | 500 | 服务器内部错误 |
| SERVICE_UNAVAILABLE | 503 | 百炼服务不可用 |

### 4.3 接口文档示例

**POST /api/generate**

**请求体**:
```json
{
  "project_idea": "我想做一个 AI 路演助手...",
  "target_user": "黑客松参与者",
  "progress": "idea",
  "pitch_duration": "1 分钟",
  "highlight": "AI 能力"
}
```

**成功响应**:
```json
{
  "success": true,
  "data": {
    "result": "# PitchForge 路演包\n\n## 一、项目结构\n..."
  },
  "message": "生成成功",
  "timestamp": "2026-06-11T00:00:00Z"
}
```

## 五、可维护性规范

### 5.1 模块化设计

**单一职责原则**: 每个模块只负责一个功能

**模块划分**:
- `api.js`: 仅负责 HTTP 请求
- `utils.js`: 仅负责工具函数
- `generateController.js`: 仅负责业务逻辑
- `bailianService.js`: 仅负责百炼 API 调用

**低耦合**: 模块间通过接口通信，不直接依赖实现

**高内聚**: 相关功能放在同一模块

### 5.2 可扩展性设计

**插件化架构**: AI 能力可替换 (百炼 → 其他平台)

**配置化**: Prompt、样式、API 地址可配置

**接口抽象**: 预留多平台支持接口

```javascript
// 服务接口抽象
class AIService {
  async generate(input) {
    throw new Error('必须实现 generate 方法');
  }
}

// 百炼实现
class BailianService extends AIService {
  async generate(input) {
    // 百炼 API 调用逻辑
  }
}

// 未来可添加其他实现
// class OpenAIService extends AIService { ... }
```

### 5.3 测试规范

**单元测试**: 核心函数测试覆盖 > 80%

**集成测试**: API 接口测试

**E2E 测试**: 用户流程测试 (输入 → 生成 → 输出)

**测试文件命名**: `*.test.js` 或 `*.spec.js`

## 七、安全规范

### 7.1 数据安全

**API Key 管理**:
- ✅ 存储在 `.env` 文件
- ✅ 添加到 `.gitignore`
- ✅ 通过环境变量注入
- ❌ 绝不硬编码在代码中
- ❌ 绝不提交到 Git

**输入验证**:
```javascript
// 前端验证
if (!projectIdea.trim()) {
  showError('请输入项目想法');
  return;
}

if (projectIdea.length > 2000) {
  showError('项目想法不能超过 2000 字');
  return;
}

// 后端验证 (双重保险)
const schema = Joi.object({
  project_idea: Joi.string().required().max(2000),
  target_user: Joi.string().optional().max(100),
  progress: Joi.string().valid('idea', '原型', 'Demo', '已上线').optional(),
  pitch_duration: Joi.string().valid('30 秒', '1 分钟', '3 分钟').optional(),
  highlight: Joi.string().optional().max(200)
});
```

**输出转义** (防 XSS):
```javascript
// 使用 textContent 而非 innerHTML
element.textContent = userInput;

// 或使用 DOMPurify 清理 HTML
import DOMPurify from 'dompurify';
element.innerHTML = DOMPurify.sanitize(userInput);
```

### 7.2 API 安全

**限流控制**:
```javascript
// Express 限流中间件
import rateLimit from 'express-rate-limit';

const limiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 分钟
  max: 10, // 最多 10 次请求
  message: '请求过于频繁，请稍后再试'
});

app.use('/api/generate', limiter);
```

**错误脱敏**:
```javascript
// ❌ 不要暴露内部错误细节
res.status(500).json({ error: err.stack });

// ✅ 返回通用错误信息
res.status(500).json({
  error: {
    code: 'INTERNAL_ERROR',
    message: '服务器内部错误，请稍后重试'
  }
});
```

**日志记录**:
```javascript
// 记录关键操作日志
logger.info('生成请求', { 
  timestamp: new Date().toISOString(),
  inputLength: input.project_idea.length,
  requestId: generateRequestId()
});

// 记录错误日志 (不含敏感信息)
logger.error('API 调用失败', {
  error: err.message,
  requestId: request.id
});
```

## 八、性能规范

### 8.1 加载性能

**首屏加载目标**: < 3 秒

**优化策略**:
1. **资源压缩**:
   - CSS/JS 使用 Gzip/Brotli 压缩
   - 图片使用 WebP 格式
   
2. **懒加载**:
   - 非关键资源延迟加载
   - 图片添加 `loading="lazy"`

3. **CDN 加速**:
   - 使用 Vercel/Netlify 全球 CDN
   - 静态资源缓存策略

### 8.2 响应性能

**API 响应目标**: < 2 秒 (不含 AI 生成时间)

**优化策略**:

## 九、版本管理规范

### 9.1 Git 提交规范

**提交信息格式**:
```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type 类型**:
- `feat`: 新功能
- `fix`: 修复 Bug
- `docs`: 文档更新
- `style`: 代码格式 (不影响功能)
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具链相关

**示例**:
```
feat(frontend): 添加路演包生成功能

- 完成输入表单验证
- 接入百炼工作流 API
- 添加 Loading 状态展示
- 实现结果渲染

Closes #123
```

### 9.2 分支管理

**主分支**: `main` (生产环境)

**开发分支**: `develop` (开发环境)

**功能分支**: `feature/功能名称`

**修复分支**: `fix/问题描述`

**工作流**:
```
feature/xxx → develop → main
                  ↓
              测试通过后合并
```
